# Diesel ORM: Schema, Queries, Migrations, Associations

## When to Use
- Type-safe database access in Rust applications
- SQL schema management with compile-time query checking
- Complex joins, aggregations, and transactional operations
- PostgreSQL, MySQL, or SQLite backends

## Standard Pattern

```rust
// === Dependencies (Cargo.toml) ===
// diesel = { version = "2.1", features = ["postgres", "r2d2", "serde_json"] }
// diesel_cli = { version = "2.1", features = ["postgres"] }  # CLI tool
// serde = { version = "1", features = ["derive"] }
// r2d2 = "0.8"


// === Schema (generated by diesel migration) ===
// diesel migration generate create_users
// - migrations/XXXX_create_users/up.sql
// - migrations/XXXX_create_users/down.sql

// --- up.sql ---
// CREATE TABLE users (
//     id          BIGSERIAL PRIMARY KEY,
//     email       VARCHAR(255) NOT NULL UNIQUE,
//     name        VARCHAR(255) NOT NULL,
//     role        VARCHAR(50) NOT NULL DEFAULT 'user',
//     created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
//     updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
// );
//
// CREATE TABLE posts (
//     id          BIGSERIAL PRIMARY KEY,
//     user_id     BIGINT NOT NULL REFERENCES users(id),
//     title       VARCHAR(500) NOT NULL,
//     body        TEXT NOT NULL,
//     published   BOOLEAN NOT NULL DEFAULT false,
//     created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
// );

// --- down.sql ---
// DROP TABLE IF EXISTS posts;
// DROP TABLE IF EXISTS users;

// Run: diesel migration run


// === Generated schema.rs (auto-generated) ===
// diesel print-schema > src/schema.rs

diesel::table! {
    users (id) {
        id -> Int8,
        email -> Varchar,
        name -> Varchar,
        role -> Varchar,
        created_at -> Timestamptz,
        updated_at -> Timestamptz,
    }
}

diesel::table! {
    posts (id) {
        id -> Int8,
        user_id -> Int8,
        title -> Varchar,
        body -> Text,
        published -> Bool,
        created_at -> Timestamptz,
    }
}

diesel::joinable!(posts -> users (user_id));
diesel::allow_tables_to_appear_in_same_query!(users, posts);


// === Models ===

use diesel::prelude::*;
use serde::{Serialize, Deserialize};
use chrono::{DateTime, Utc};

#[derive(Debug, Queryable, Selectable, Serialize)]
#[diesel(table_name = crate::schema::users)]
#[diesel(check_for_backend(diesel::pg::Pg))]
struct User {
    id: i64,
    email: String,
    name: String,
    role: String,
    created_at: DateTime<Utc>,
    updated_at: DateTime<Utc>,
}

#[derive(Debug, Insertable)]
#[diesel(table_name = crate::schema::users)]
struct NewUser {
    email: String,
    name: String,
}

#[derive(Debug, Queryable, Selectable, Associations, Serialize)]
#[diesel(table_name = crate::schema::posts)]
#[diesel(belongs_to(User))]
#[diesel(check_for_backend(diesel::pg::Pg))]
struct Post {
    id: i64,
    user_id: i64,
    title: String,
    body: String,
    published: bool,
    created_at: DateTime<Utc>,
}


// === Connection Pool ===

use diesel::r2d2::{self, ConnectionManager};

type DbPool = r2d2::Pool<ConnectionManager<PgConnection>>;

fn create_pool(database_url: &str) -> DbPool {
    let manager = ConnectionManager::<PgConnection>::new(database_url);
    r2d2::Pool::builder()
        .max_size(10)
        .build(manager)
        .expect("Failed to create database pool")
}


// === Queries ===

fn get_user(pool: &DbPool, user_id: i64) -> Result<User, diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().map_err(|e| {
        diesel::result::Error::DatabaseError(
            diesel::result::DatabaseErrorInformation::new("ConnectionError", &e.to_string()),
        )
    })?;

    users.find(user_id).first(&mut conn)
}

fn find_users_by_role(pool: &DbPool, role_name: &str) -> Result<Vec<User>, diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().unwrap();
    users
        .filter(role.eq(role_name))
        .order(created_at.desc())
        .load::<User>(&mut conn)
}

fn create_user(pool: &DbPool, email: &str, name: &str) -> Result<User, diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().unwrap();
    let new_user = NewUser {
        email: email.to_string(),
        name: name.to_string(),
    };

    diesel::insert_into(users)
        .values(&new_user)
        .returning(User::as_returning())
        .get_result(&mut conn)
}

fn get_user_with_posts(
    pool: &DbPool,
    user_id: i64,
) -> Result<(User, Vec<Post>), diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().unwrap();
    let user = users.find(user_id).first::<User>(&mut conn)?;
    let posts = Post::belonging_to(&user)
        .filter(crate::schema::posts::published.eq(true))
        .load::<Post>(&mut conn)?;

    Ok((user, posts))
}


// === Transactions ===

fn transfer_post_ownership(
    pool: &DbPool,
    post_id: i64,
    new_owner_id: i64,
) -> Result<Post, diesel::result::Error> {
    use crate::schema::posts::dsl::*;

    let mut conn = pool.get().unwrap();

    conn.transaction(|conn| {
        // Verify post exists
        let post = posts.find(post_id).first::<Post>(conn)?;

        // Update owner
        diesel::update(posts.find(post_id))
            .set(user_id.eq(new_owner_id))
            .returning(Post::as_returning())
            .get_result(conn)
    })
}


// === Updates and Deletes ===

fn update_user_role(
    pool: &DbPool,
    user_id: i64,
    new_role: &str,
) -> Result<User, diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().unwrap();

    diesel::update(users.find(user_id))
        .set(role.eq(new_role))
        .returning(User::as_returning())
        .get_result(&mut conn)
}

fn delete_user(pool: &DbPool, user_id: i64) -> Result<usize, diesel::result::Error> {
    use crate::schema::users::dsl::*;

    let mut conn = pool.get().unwrap();
    diesel::delete(users.find(user_id)).execute(&mut conn)
}
```

## Common Mistakes

```rust
// WRONG: Calling .load() without specifying type (compile error)
let users = users_dsl::load(&mut conn);  // Can't infer type

// CORRECT: Specify the return type or turbofish
let users: Vec<User> = users_dsl::load(&mut conn)?;
// Or: users_dsl::load::<User>(&mut conn)?;


// WRONG: Forgetting .get_result() for single row (compile error)
let user = users.find(1).execute(&mut conn);  // execute returns usize

// CORRECT: Use .first() or .get_result()
let user: User = users.find(1).first(&mut conn)?;


// WRONG: Not using returning() with PostgreSQL (compile error on INSERT)
diesel::insert_into(users).values(&new_user).execute(&mut conn);
// Doesn't return the inserted row! Need a second query.

// CORRECT: Use returning() for inserted data
diesel::insert_into(users)
    .values(&new_user)
    .returning(User::as_returning())
    .get_result(&mut conn)?;


// WRONG: Running migrations in application code without checking
// Migrations should be run by CLI or startup script, not main app

// CORRECT:
// Diesel CLI: diesel migration run
// Or embedded: embed_migrations!() + run at startup once


// WRONG: Not using connection pool (opening connections per request)
let conn = PgConnection::establish(&url).unwrap();  // Expensive!

// CORRECT: Use r2d2 connection pool
let conn = pool.get().unwrap();
```

## Gotchas
- **Compile-time checking requires schema.rs regeneration**: After ANY schema change (adding columns, tables), run `diesel print-schema` to regenerate `schema.rs`. The generated code is the truth — your query code won't compile if it's out of sync.
- **Connection pool sizing**: Diesel's r2d2 pool is synchronous during checkout (`pool.get()` blocks). Set `max_size` based on your async runtime's thread pool size. For tokio with 4 worker threads, 4-8 connections is usually sufficient.
- **PostgreSQL vs SQLite feature flags**: Features like `RETURNING`, `ON CONFLICT`, and `TIMESTAMPTZ` are backend-specific. Code written for PostgreSQL may not compile with the SQLite backend. Use `#[cfg(feature = "postgres")]` for backend-specific code.
- **Serde integration**: `diesel` and `serde` are separate. To serialize query results, add `#[derive(Serialize)]` on `Queryable` structs. For JSON columns, use `serde_json::Value` with Diesel's `serde_json` feature.
- **Boxed queries for dynamic filtering**: When building queries with conditional `WHERE` clauses, use `.into_boxed()` to return a dynamically-typed query. Without it, each `filter()` call changes the type, making conditional building impossible.
- **Nullable columns and Option**: Diesel maps nullable columns to `Option<T>`. If a column is `NOT NULL` in the schema but you use `Option<T>`, Diesel will still work but panics at runtime if the value is actually NULL. Keep types in sync with the schema.
- **Migration numbering**: Diesel uses timestamp-prefixed migration directories (e.g., `20250523000001_create_users`). The prefix determines migration order. If multiple developers create migrations with the same prefix, you'll get conflicts. Coordinate prefixes or use sequential numbers.
